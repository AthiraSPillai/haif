# HAIF Adoption Playbook

Start small. Do not turn HAIF into ceremony.

## Week 1

- Add `.haif/records` to one repo or project workspace.
- Convert five active Jira tickets into HAIF proposals, designs, or decisions.
- Convert two important decisions into HAIF decision records.

## Week 2

- Ask agents to run HAIF preflight before planning or coding.
- Mark agent-created tickets as proposed until reviewed.
- Start tracking conflicts and superseded docs.

## Week 3

- Add GitHub PR checklist.
- Require design review for architecture-level changes.
- Export context for agents before implementation.

## Week 4

- Review duplicate tickets, stale docs, unresolved conflicts, and design drift.
- Keep the lifecycle states that helped.
- Remove any steps that created busywork.
