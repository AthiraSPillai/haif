# HAIF Lifecycle

HAIF separates possibility from commitment.

```text
Signal -> Proposal -> Intent -> Design -> Decision -> Task -> Implementation -> Review -> Release
```

## States

- `Signal`: an observation, risk, issue, or opportunity.
- `Proposal`: suggested work that is not yet accepted.
- `Intent`: accepted problem or opportunity with owner and scope.
- `Design`: proposed direction.
- `Decision`: approved, rejected, or superseded direction.
- `Task`: execution work derived from accepted intent or decision.
- `Implementation`: active work.
- `Review`: human review checkpoint.
- `Release`: final shipped state and updated source of truth.

## Rule

AI agents can propose and execute. Humans approve commitment, scope, design, and release readiness.
